Diagram style: one diagram, two ways to draw it

The same hub-and-spoke concept diagram (an LLM-inference memory footprint), rendered two ways. Watch: ① how the colour-coded resource pills read, ② whether the hand-drawn style still holds up in the light theme. Use the toggle, top right.

HBM / on-GPU memory DRAM main memory SSD · NAND storage CPU DPU data movement

Option 1 Clean: geometric strokes + resource pills

Clean geometric strokes, hub-and-spoke, whitespace, bidirectional arrows, a colour-coded pill under each box. Holds in both themes, zero dependencies.

Pros: clear in light and dark, no dependencies, single file, works for data-dense diagrams too. This is the recommended default.

Option 2 rough.js hybrid: hand-drawn outlines + clean text

rough.js draws hand-drawn boxes and lines (dark-first); the text stays in a clean font. Toggle to light to see whether the "blackboard" feel survives.

Cost: needs rough.js (runtime JS generates the squiggly paths, which breaks the single-file-static rule), the charm drops sharply in light mode, and data-dense diagrams lose precision. Best for: an English, conceptual, dark-first single diagram.
